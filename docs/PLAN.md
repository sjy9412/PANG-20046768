# PANG 개발 계획 (PLAN.md)

## 개요

PANG 게임의 구현을 5개의 Phase로 나누어 진행한다.
**각 Phase가 끝나면 고객님이 직접 브라우저에서 실행하고 피드백을 주시는 방식으로 진행된다.**
모든 Phase는 이전 Phase의 결과물 위에 쌓이며, 매 Phase마다 실제로 동작하는 게임을 확인하실 수 있다.

---

## Phase 1 — 캐릭터와 버블이 움직이는 화면

### 이번 Phase에서 만드는 것

게임의 가장 기초가 되는 뼈대를 구성한다. 아직 "게임"이라기보다는 움직이는 오브젝트들을 보여주는 단계다.

| 구현 항목 | 설명 |
|-----------|------|
| 게임 화면(Canvas) 구성 | 브라우저에서 게임 영역이 표시된다 |
| 플레이어 이동 | ← → 방향키로 캐릭터가 좌우로 움직인다 |
| 버블 등장 및 물리 이동 | 버블이 화면에 등장하여 포물선을 그리며 튀어다닌다 |
| 벽·바닥 반사 | 버블이 벽과 바닥에 부딪혀 방향을 바꾼다 |

### 고객님께서 확인해 주실 사항

> **"움직임이 자연스러운가?"** 가 핵심입니다.

- 캐릭터가 키 입력에 즉각적으로 반응하는지
- 버블이 통통 튀는 물리감이 자연스러운지 (너무 빠르거나 느리지 않은지)
- 버블이 벽/바닥에서 올바르게 반사되는지

---

## Phase 2 — 와이어를 쏘아 버블을 터뜨릴 수 있는 상태

### 이번 Phase에서 만드는 것

실제로 "플레이"가 가능한 최소 단위의 게임이 완성된다. Stage 1 한 판을 끝까지 플레이할 수 있다.

| 구현 항목 | 설명 |
|-----------|------|
| 와이어(훅) 발사 | Space 키로 위쪽으로 와이어를 발사한다 |
| 버블 분열 | Large → Medium 2개 → Small 2개 → 소멸 순서로 분열한다 |
| Block 장애물 | 화면 내 Block이 배치되고, 버블이 Block 위로 튕겨 오른다 |
| 피격 판정 | 버블에 닿으면 캐릭터가 데미지를 입는다 |
| 목숨 시스템 | 목숨 3개에서 시작, 0이 되면 게임 오버 |
| 제한 시간 | 60초 타이머가 동작하고, 시간 초과 시 목숨 감소 |
| 스테이지 클리어 | 모든 버블을 제거하면 클리어 메시지가 표시된다 |

### 고객님께서 확인해 주실 사항

> **"게임의 기본 규칙이 올바르게 동작하는가?"** 가 핵심입니다.

- 와이어를 발사했을 때 버블이 올바르게 분열되는지
- 버블이 Block 위에서 자연스럽게 튕기는지
- 캐릭터가 버블에 닿았을 때 목숨이 감소하는지
- 시간이 초과되었을 때 목숨이 감소하고 스테이지가 재시작되는지
- 버블을 전부 제거했을 때 클리어가 뜨는지

---

## Phase 3 — Mission 1의 5개 스테이지를 처음부터 끝까지 플레이

### 이번 Phase에서 만드는 것

Stage 1부터 Stage 5까지 순서대로 플레이할 수 있는 Mission 1 전체가 완성된다.

| 구현 항목 | 설명 |
|-----------|------|
| Stage 1~5 구성 | 각 스테이지의 버블 배치 및 Block 배치가 정의된다 |
| 스테이지 전환 | 클리어 후 자동으로 다음 스테이지로 넘어간다 |
| 점수 시스템 | 버블 크기별 점수 + 클리어 시 남은 시간 보너스 점수가 계산된다 |
| 게임 오버 화면 | 목숨을 모두 소진하면 게임 오버 화면이 표시되고 재시작할 수 있다 |
| Mission 클리어 화면 | Stage 5 클리어 시 미션 완료 화면이 표시된다 |

### 고객님께서 확인해 주실 사항

> **"5개 스테이지의 난이도 흐름이 자연스러운가?"** 가 핵심입니다.

- Stage 1이 충분히 쉽고 입문용으로 적합한지
- Stage 4~5에서 Block 등장 이후 난이도가 적절히 올라가는지
- 스테이지를 클리어할 때 점수가 올바르게 집계되는지
- 게임 오버 후 재시작이 잘 동작하는지
- 전체 5스테이지를 연속 플레이할 때 흐름이 자연스러운지

---

## Phase 4 — 아이템과 HUD가 추가된 완성형 게임

### 이번 Phase에서 만드는 것

게임 경험을 풍부하게 만드는 아이템 시스템과 화면 UI가 완성된다. 이 Phase부터 "게임답게" 보이기 시작한다.

| 구현 항목 | 설명 |
|-----------|------|
| 아이템 드롭 및 효과 | 더블 훅 / 다이너마이트 / 시계(버블 정지) / 방어막 4종 |
| HUD 표시 | 화면 상단에 점수 / 남은 시간 / 목숨 수 / 스테이지 번호 표시 |
| 메인 화면 | 타이틀 로고와 1P GAME / HIGH SCORE 메뉴가 있는 시작 화면 |
| 스테이지 시작 연출 | 각 스테이지 시작 시 스테이지 번호가 화면에 잠깐 표시된다 |

### 고객님께서 확인해 주실 사항

> **"아이템이 게임을 재미있게 만드는가? UI가 직관적인가?"** 가 핵심입니다.

- 아이템을 획득했을 때 효과가 눈에 띄게 나타나는지
- 각 아이템 효과가 PRD에 정의된 대로 동작하는지 (더블 훅, 다이너마이트, 시계, 방어막)
- HUD의 점수·시간·목숨 정보가 실시간으로 올바르게 갱신되는지
- 메인 화면에서 게임 시작까지의 흐름이 자연스러운지

---

## Phase 5 — 최종 완성본 (사운드 · 마무리)

### 이번 Phase에서 만드는 것

사운드, 최고 점수 저장 등 완성도를 높이는 마지막 요소들이 추가된다.

| 구현 항목 | 설명 |
|-----------|------|
| 사운드 이펙트 | 버블 터지는 소리, 피격 소리, BGM 등 |
| HIGH SCORE 저장 | 최고 점수가 브라우저에 저장되어 다음 접속 시에도 유지된다 |
| 버그 수정 및 밸런스 조정 | 이전 Phase 피드백을 반영한 최종 조정 |

### 고객님께서 확인해 주실 사항

> **"게임이 완성된 제품으로서 손색이 없는가?"** 가 핵심입니다.

- 사운드가 게임의 몰입감을 높이는지 (또는 불필요하게 거슬리지 않는지)
- HIGH SCORE가 브라우저를 닫았다 다시 열어도 유지되는지
- 전체적으로 플레이하며 불편하거나 어색한 부분이 없는지

## Phase 요약

| Phase | 핵심 결과물 | 고객님 확인 포인트 |
|-------|------------|-------------------|
| Phase 1 | 캐릭터 이동 + 버블 물리 | 움직임의 자연스러움 |
| Phase 2 | 와이어 발사 + 분열 + 목숨/타이머 | 기본 규칙의 정확성 |
| Phase 3 | Mission 1 전체 5스테이지 | 난이도 흐름과 점수 체계 |
| Phase 4 | 아이템 4종 + HUD + 메인 화면 | 아이템 재미 + UI 직관성 |
| Phase 5 | 사운드 + HIGH SCORE | 완성도 전반 |
